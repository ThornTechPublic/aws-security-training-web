Appendix: Create an EC2 Key Pair

Create an EC2 Key Pair

Since you will be connecting to EC2 instances over SSH, you need an EC2 key pair. The Key Pair is a public key (you download the private key upon creation). When you provision an instance, AWS injects the public key. This way, you can connect securely on first launch.

  • Go to Services > EC2
  • On the left, under NETWORK & SECURITY, click on Key Pairs
  • Click Create Key Pair
  • Enter a Key pair name (use your name so it's easy to identify whose key it is)

  • Click Create
  • Your browser will download a file named <key-name>.pem, which is your private key

Key Preparation for PC

In this section, you will convert the private key to a format that is compatible with PuTTY.

To download PuTTY, visit the link below. If you don't have permission to install software on your machine, you can try running the portable .exe

https://www.chiark.greenend.org.uk/~sgtatham/putty/latest.html

PuTTY does not natively support the PEM format that AWS uses, so you need to first convert your .PEM file to a .PPK file, which stands for PuTTY Private Key. To do this, you use the PuTTY Key Generator utility packaged with PuTTY.

  • Open PuTTYgen
  • Click the Load button

  • Browse to the location where you downloaded your .PEM key.
  • When browsing for the key, be sure to select All Files in the dropdown menu.

  • Click Open when you have selected your key.
  • You should see the following success dialog. Click OK.

  • Optional: Enter a Key passphrase and Confirm passphrase.
  • Click Save private key. This will save the PPK file.

  • If prompted, click Yes to save without a passphrase

  • Finally, save the .ppk file

Key preparation for MacOS or Linux

In this section, you will restrict the permissions on the private key.

As a security feature in Linux, your private key can only be writable by the owner. Otherwise, SSH commands will produce an error.

Open Terminal (use the Command-Space shortcut, and search for Terminal)

Find the private key from the EC2 Key Pair that you downloaded earlier

cd ~/Downloads/
ls -ltr

Restrict permissions on the private key.

chmod 600 key-pair-private-key.pem
