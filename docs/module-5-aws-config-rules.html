Module 5: AWS Config Rules

Objective

In this lab, you will use AWS Config Rules to verify that no S3 buckets are publicly accessible. There is one S3 bucket that is public, which will show up on the list of non-compliant resources. You will remove public access to this bucket, and bring the Config Rule into compliance.

The goal of this lab is to:

  • Familiarize yourself with the AWS Config Rules console.
  • Get some practice managing S3 bucket permissions

Spin up the CloudFormation template

Spin up the CloudFormation template using these instructions: Spin up a CloudFormation template

  • CloudFormation template: ConfigS3.template
  • CloudFormation Parameters:
    • S3PublicBucketName: Enter a unique S3 bucket name
  • Typical provisioning duration: 2 minutes

Test S3 public access

The CloudFormation template provisions a public S3 bucket. In this section, you will upload a file and verify that you can access its public URL.

In the S3 console, locate the bucket that you named earlier in the CloudFormation parameters. You should see a Public tag in the Access column.

Click on the bucket name.

Click Upload

Drag and drop a file (preferably an image) in the center area, and click Upload

From the list of S3 objects, click on the file that you uploaded.

Toward the bottom of the S3 object Overview tab, right-click on the Object URL and open it in a new tab. You should be able to see the file contents.

The Object URL a way to simulate unauthenticated access, since it doesn't use cookies or tokens.

Evaluate Config Rule Compliance Status

In this section, you will evaluate the status of your Config rule.

In the AWS console, search for Config among the list of services. This should take you to the AWS Config dashboard.

On the left, click on Rules

In the Rule Name list, click on the Config rule that you created using CloudFormation.

In the Rule details page, note the name of any S3 buckets toward the bottom that are causing non-compliance.

Remove public access from the S3 bucket

In this section, you will lock down the S3 bucket so that it is no longer public.

Go back to the S3 console and select your public S3 bucket.

Click on the Permissions tab.

Click on the Bucket Policy sub-tab.

Click Delete, and confirm Delete when prompted.

For good measure, you will enable a setting that acts as a child safety lock for preventing accidental public S3 access.

Click on the Block public access button, and click on Edit

Check the box next to Block all public access, and then click Save

When prompted, type in "confirm" and then click Confirm

Click back to the Bucket Policy sub-tab. You should see that it won't let you make any changes to the bucket policy.

Verify that the S3 object is private

Go back to the S3 bucket Overview tab.

Click on your S3 object, and click the Object URL.

Instead of opening the file, you should now get an error message:

Verify compliance with AWS Config

In this last section, you will confirm that the S3 bucket complies with the AWS Config rule.

In the AWS console, go back to AWS Config Rules.

Select your rule from the list.

On the top right, click on Re-evaluate

It might take a few minutes, but your S3 bucket should show a status of Compliant

Note: The AWS Config rule may not reach compliance until all S3 buckets in your AWS account are private. This could depend on how far along the lab other members of your team are.
