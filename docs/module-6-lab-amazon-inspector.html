Module 6: Lab Amazon Inspector

Objectives

  • Become familiar with scanning applications with Amazon Inspector

Overview

Amazon Inspector performs security assessments on EC2 instances. It comes with various rules packages like CVE and CIS. Scans can be run ad hoc or on a scheduled basis. Report results include recommendations on how to fix issues.

In this lab, you will scan a WordPress server that you created in a previous lab. Scans can take a long time, depending on the rules packages and specified duration. In the interest of time, you will only run the Network Reachability package and choose the shortest duration. This way, you can at least see your own results. To see a sample scan with all the rules packages, refer to the PDF for this section.

Amazon Inspector relies on an agent installed on the EC2 instance (for packages other than Network Reachability). Conveniently, the WordPress server from the previous lab installs this agent in the EC2 UserData section. Otherwise, you can install the agent using SSM, or baking it into a golden AMI. Agent installation is not covered in this lab.

Initiate a Scan

  • In the Services drop-down, search for Inspector
  • If you see the first time screen, click Get Started.
  • Otherwise, click Help me create an Assessment

  • Leave the boxes checked for Network Assessments and Host Assessments

  • Click Advanced Setup

Define an assessment target

  • Name: enter a unique name
  • All Instances: Uncheck the box
  • Tags: Enter the following:
    • Key: aws-cloudformation:stack-name
    • Value: From the drop-down, pick the name of your WordPress CloudFormation stack
  • Install Agents: Keep the box checked

  • Click Next

Define an Assessment Template

  • Name: Enter a unique name
  • Rules packages: Click the x to remove everything except for Network Reachability-1.1
  • Duration: Select 15 Minutes
  • Assessment schedule: Uncheck the box

  • Click Next

Review Page

Click Create

Check the status

  • Toward the left, click on Assessment runs
  • Hit the refresh icon on the right, until you find your run on the list
  • Expand the disclosure triangle. You should see something like this:

  • Toward the bottom, click on Show AWS agents
  • Here, you can check the status of any installed agents

Amazon Inspector should run for 15 minutes. In the meantime, you can review the PDF that comes with this lab to see some sample findings.

Review findings

Click on the number of Findings, if there are any

Examine the findings, which are mainly related to ports accessible to the internet.
