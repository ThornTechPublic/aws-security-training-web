Appendix: Spin Up a CloudFormation Template

CloudFormation is an orchestration tool for provisioning infrastructure using code. In other words, you can spin up complicated infrastructure from a template.

We use CloudFormation heavily in the labs. Setting everything up manually from the AWS console is too time consuming and error prone. Also, you can copy the syntax from these templates to build your own test scenarios later on.

  • Go to Services > CloudFormation
  • Click Create stack

Create stack page

  • Click Upload a template file
  • Click Choose file
    • Select the template file specified in the lab
  • Click Next

Specify stack details page

  • Fill out the fields according to the Lab's instructions. Here are some common field types:
    • Stack name: Specify a unique name for your CloudFormation stack
    • EC2Type: Keep this as the default t3.micro
    • InputCIDR: Some of the templates ask for your current IP address to allow ingress to an EC2 instance. Visit http://checkip.dyndns.org/ to get your current public IP address. Then, append /32 to your IP address. For example: 69.143.221.67/32
    • KeyPair: If an EC2 instance is involved, choose your KeyPair from the drop-down list.
    • VPCIPRange: From the drop-down list, pick an IP range that nobody else in your group is using. This is to avoid confusion when troubleshooting.
  • When finished, click Next

Configure stack options page

  • Optional: Enter some Tag information. This makes troubleshooting easier.
    • Key: Name
    • Value: <your name>

  • When finished, click Next

Review page

  • Scroll to the bottom. If your CloudFormation stack creates an IAM resource, check the box in the Capabilities section.

  • Click Create stack

Wait for the Stack to create

Click on the Events tab.

You should see various events as CloudFormation provisions different Resources.

Toward the right, click on the Refresh icon to see new events.

The Lab should indicate approximately how long the stack will take to spin up.
