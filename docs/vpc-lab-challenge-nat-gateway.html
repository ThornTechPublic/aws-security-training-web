Module 2 Challenge: NAT Gateway

Objective

  • Learn how to provision a NAT Gateway to provide outbound internet access from a private subnet

Overview

The instances in your Private Subnet cannot be directly accessed from the public internet, and this gives you a layer of protection.

But, what if you need to access internet resources? For example, you might want to run yum updates or download a Git repo.

In this challenge, you need to:

  • Give your private EC2 instance outbound internet access. For example, the following command should work without timing out:
sudo yum update

The answer is to add a NAT Gateway. This gives instances in your Private Subnet outbound internet access, without exposing inbound access.

Challenge

In the lab, you are given step-by-step instructions on what to do.

In the challenge, you will be given high level instructions, and it's up to you to poke around the AWS console on your own to accomplish these tasks. The hardest part of the challenge is finishing in the time remaining — if there is any — so keep an eye on the clock.

High level instructions

To wire everything up properly, you need to do the following:

  • Provision an Elastic IP address (the NAT Gateway needs one)
  • Add a NAT Gateway to the Public Subnet
  • Give the Elastic IP address to the NAT Gateway
  • Point all non-local traffic on the Private Route Table to the NAT Gateway

You can explore the AWS console and figure out how to accomplish these steps in the time remaining.

Alternatively, you can integrate the following code into your existing CloudFormation template, and then perform a CloudFormation stack update:

  NATIPAddress:
    Type: AWS::EC2::EIP
    DependsOn: AttachGateway
    Properties:
      Domain: vpc
  NATGateway:
    DependsOn:
      - AttachGateway
      - NATIPAddress
    Type: AWS::EC2::NatGateway
    Properties:
      AllocationId: !GetAtt NATIPAddress.AllocationId
      SubnetId: !Ref PublicSubnet
  PrivateDefaultRoute:
    Type: AWS::EC2::Route
    Properties:
      DestinationCidrBlock: 0.0.0.0/0
      NatGatewayId: !Ref NATGateway
      RouteTableId: !Ref PrivateRouteTable

Check your work

If your Private EC2 instance is able to perform a yum update without timing out, you have successfully completed the challenge.
